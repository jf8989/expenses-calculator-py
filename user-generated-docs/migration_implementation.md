# Migration Implementation

This file tracks the migration steps. 

## Instructions
Every time a phase is completed, the `pulse.md` file at root must be updated or created if it doesn't exist. Add a summary of what was done as a simple snapshot.

## Phases
1. **Branching & Cleanup**: Create branch and move Python code to legacy folder.
2. **Template Integration**: Clone and set up the Next.js template.
3. **Environment Setup**: Create `.env.example`.
4. **Auth Migration**: Implement Firebase auth in Next.js.
5. **Logic Migration**: Port participant and session logic.
6. **UI Enhancement**: Polishing the frontend.
